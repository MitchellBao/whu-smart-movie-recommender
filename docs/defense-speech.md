# 答辩讲稿与演示流程

本文档用于课程答辩。前半部分是口头讲稿，后半部分是现场演示步骤。讲述时不需要逐字照读，可以按每节要点展开。

## 一、口头讲稿

### 1. 开场

各位老师好。我们组的项目是“智能电影推荐系统”。项目基于 MovieLens 数据集，实现了电影数据导入、用户评分、个性化推荐、推荐解释、用户画像和可视化展示。

我是鲍明頡，主要负责项目整体架构、数据库设计、后端主链路、前后端联调、DeepSeek 接入、数据可视化和最终整合。

### 2. 项目要解决的问题

电影数量很多，用户很难从大量候选电影中快速找到自己可能喜欢的内容。我们的系统围绕这个问题做了三件事：

1. 根据用户评分生成推荐列表。
2. 给出推荐理由，让用户知道为什么推荐。
3. 允许用户继续评分、收藏、标记不感兴趣，让推荐形成反馈闭环。

项目主链路如下：

```text
MovieLens 数据 -> MySQL -> Python 推荐算法 -> Spring Boot 后端 -> Vue 前端
```

DeepSeek 只负责解释和问答，不直接参与推荐排序。

### 3. 系统架构

系统分为四层：

| 层次 | 技术 | 作用 |
|---|---|---|
| 前端 | Vue 3 / Vite | 登录、电影浏览、评分、推荐展示、画像、问答 |
| 后端 | Spring Boot | 统一 API、业务编排、数据库访问、LLM 调用 |
| 算法服务 | Python FastAPI | 读取评分数据并计算推荐 |
| 数据层 | MySQL | 保存用户、电影、评分、推荐缓存和显式偏好 |

前端不会直接访问算法服务和 DeepSeek，统一请求 Spring Boot 后端。这样做的好处是接口清晰，后续部署和维护也更方便。

### 4. 数据库设计

核心表如下：

| 表 | 作用 |
|---|---|
| `users` | 保存用户信息 |
| `movies` | 保存 MovieLens 电影信息 |
| `ratings` | 保存用户评分，是推荐算法的主要输入 |
| `recommendations` | 保存推荐缓存和推荐理由 |
| `user_movie_preferences` | 保存想看、收藏、不感兴趣 |

其中 `ratings` 负责记录用户行为，`recommendations` 负责保存推荐结果，`user_movie_preferences` 用来补充显式反馈。

### 5. 推荐算法流程

算法服务直接读取 MySQL 中的 `ratings` 表。流程是：

1. 读取用户、电影、评分三类数据。
2. 构造用户-电影评分矩阵。
3. 结合相似用户偏好和 SVD 隐语义特征生成推荐。
4. 如果用户数据不足，使用热门高分电影兜底。
5. 返回电影 ID、推荐分和算法解释给后端。

这里需要说明一点：推荐结果中的 `score` 是排序分，不等同于用户评分。用户评分是 `0.5` 到 `5.0`，推荐分只用于比较推荐优先级。

### 6. 用户反馈闭环

系统支持两类反馈。

第一类是评分反馈：

- 用户可以给电影打分。
- 可以修改评分。
- 可以删除评分。
- 评分变化后会刷新推荐。

第二类是显式偏好反馈：

- 想看。
- 收藏。
- 不感兴趣。

其中“不感兴趣”会影响推荐展示。用户标记后，前端会立即移除该电影，后端后续读取推荐时也会过滤它。

### 7. DeepSeek 问答设计

DeepSeek 在本项目中只作为解释层使用。它不负责推荐排序，推荐排序仍由算法服务完成。

后端调用 DeepSeek 前会整理一份上下文，包括：

- 当前推荐列表及排名。
- 推荐分和推荐标签。
- 用户高分电影。
- 用户低分电影。
- 想看、收藏、不感兴趣列表。

所以用户问“当前推荐第一名为什么适合我”时，DeepSeek 能知道第一名是哪部电影；用户问“根据我的评分推荐一部电影”时，也能看到真实评分摘要。

如果没有配置 DeepSeek Key，系统会返回离线默认解释，推荐主流程仍然可以运行。

### 8. 前端功能

前端分为六个主要菜单：

| 菜单 | 功能 |
|---|---|
| 推荐结果 | 展示推荐列表、推荐分、结构化理由、可信度标签 |
| 电影库与评分 | 搜索、分页、筛选、电影详情、提交评分 |
| 我的评分 | 评分统计、排序筛选、快速改分、删除评分 |
| 我的画像 | 展示评分偏好、想看、收藏、不感兴趣 |
| 数据看板 | 展示数据规模、类型分布、热门电影、推荐分布 |
| 智能问答 | 使用 DeepSeek 解释推荐和回答观影问题 |

### 10. 总结

这个项目的重点不是单独做一个推荐接口，而是把数据、算法、后端、前端、解释和反馈连成一条完整链路。

目前系统已经能完成：

- 从 MovieLens 到 MySQL 的数据导入。
- 从用户评分到推荐结果的计算。
- 从推荐结果到前端展示。
- 从用户反馈到推荐刷新。
- 从推荐结果到 DeepSeek 解释。

后续如果继续完善，主要方向是密码哈希、权限控制、数据库迁移脚本、推荐算法离线评估和前端组件拆分。

## 二、现场演示流程

### 1. 启动项目

项目根目录启动算法服务和后端：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\deploy\scripts\start-local-fullstack.ps1
```

另开终端启动前端：

```powershell
cd frontend
npm install
npm run dev
```

访问：

```text
http://127.0.0.1:5173/
```

### 2. 登录或注册

演示点：

- 用户登录后，系统自动使用当前用户 ID。
- 后续推荐、评分、画像、问答都围绕该用户展开。

### 3. 电影库与评分

操作：

1. 进入“电影库与评分”。
2. 搜索 `Matrix`。
3. 点击电影查看详情。
4. 展示评分人数、平均评分、评分分布、相似电影。
5. 给电影打分，例如 `4.5`。

讲解点：

- 评分写入 `ratings` 表。
- 评分会触发推荐刷新。
- 评分范围是 `0.5` 到 `5.0`，步长是 `0.5`。

### 4. 推荐结果

操作：

1. 进入“推荐结果”。
2. 查看推荐列表。
3. 展示推荐分、推荐理由、可信度标签。
4. 点击“不感兴趣”。

讲解点：

- 推荐分是排序分，不是用户评分。
- 结构化理由来自后端对用户偏好、电影评分和算法分数的整理。
- “不感兴趣”会进入 `user_movie_preferences`，后续推荐会过滤。

### 5. 我的评分

操作：

1. 进入“我的评分”。
2. 展示评分数量、平均分、高频类型、最近评分时间。
3. 使用排序和筛选。
4. 修改一条评分。
5. 删除一条评分。

讲解点：

- 我的评分是用户画像的数据来源。
- 评分修改和删除都会影响后续推荐。

### 6. 我的画像

操作：

1. 进入“我的画像”。
2. 展示评分偏好。
3. 展示想看、收藏、不感兴趣。

讲解点：

- 这里把隐式反馈和显式反馈集中展示。
- 画像用于解释系统为什么会给出某些推荐。

### 7. 数据看板

操作：

1. 进入“数据看板”。
2. 展示电影数、用户数、评分数。
3. 展示类型分布、热门电影、评分分布、推荐分布。

讲解点：

- 看板用于说明数据规模和推荐结果特征。
- 图表不参与推荐计算，只用于辅助理解。

### 8. 智能问答

操作：

1. 进入“智能问答”。
2. 查看“当前会参考”的上下文。
3. 点击模板“解释当前推荐第一名为什么适合我”。
4. 再问“根据我的评分推荐一部电影”。

讲解点：

- 前端传 `userId`、`queryText`、`topN`。
- 后端会把推荐排名、评分摘要、想看、收藏、不感兴趣传给 DeepSeek。
- DeepSeek 负责解释，不参与排序。

### 9. 自动测试

答辩前可以执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\deploy\scripts\test-local-fullstack.ps1 -SkipLlmCheck
```

看到以下结果即可：

```text
Total: 18, Passed: 18, Failed: 0
```

