# 第一部分 选题及团队组建报告

## 1.1 选题目的和意义

随着数字内容规模的持续增长，用户在海量信息中高效发现符合自身兴趣的内容已成为信息系统设计中的重要问题。推荐系统通过对用户历史行为数据的分析与建模，实现个性化的信息过滤与内容分发，已成为视频平台、电商平台以及内容社区等互联网服务中的关键技术。

本项目旨在设计并实现一个面向大规模用户行为数据的智能电影推荐系统。系统以电影推荐作为实验场景，通过用户评分数据、电影元数据和推荐算法模型，生成符合用户兴趣的个性化电影推荐结果。同时，系统引入大语言模型能力，对推荐结果进行自然语言解释，使用户不仅能够看到“推荐什么”，也能够理解“为什么推荐”。

本项目不仅关注推荐结果本身，也关注推荐系统内部的数据处理、算法实现和工程架构。系统以 MovieLens 等公开数据集为基础，构建从数据清洗、评分整理、特征建模、推荐计算到结果展示的完整流程。在算法层面，项目围绕协同过滤、矩阵分解等经典推荐方法展开；在系统层面，项目采用前后端分离、后端服务编排、算法微服务和数据库持久化结合的架构；在智能增强层面，项目使用大语言模型生成推荐理由，提升推荐结果的可解释性和交互体验。

在应用层面，电影推荐场景具有数据公开、结构清晰、用户理解成本低等特点，适合作为课程综合实践项目。系统整体采用通用 User-Item 推荐模型思想，后续在替换数据源和特征配置后，也可扩展到图书推荐、音乐推荐或电商商品推荐等其他应用场景。

## 1.2 研究内容

本项目研究内容主要包括数据处理、推荐算法、系统工程和智能解释四个方面。

### 1.2.1 数据处理

系统以 MovieLens 等公开数据集作为实验数据来源，围绕用户评分记录、电影标题、电影类型等数据建立基础数据模型。数据处理内容包括：

- 整理用户、电影、评分等基础实体。
- 构建用户-电影评分关系。
- 为推荐算法提供结构化输入。
- 通过数据库表结构支持推荐结果和推荐理由的持久化存储。

### 1.2.2 推荐算法

项目围绕经典推荐算法展开设计，重点包括：

- 基于用户行为的协同过滤思想。
- 基于评分矩阵的候选电影生成。
- 使用 SVD 等矩阵分解方法提取隐语义特征。
- 根据预测得分生成 Top-N 推荐结果。

推荐算法服务采用 Python 实现，并通过 HTTP 接口向 Java 后端提供推荐计算能力。

### 1.2.3 系统工程

系统采用分层架构与前后端分离设计。整体划分为应用层、服务层、算法层和数据层：

- 应用层由 Vue 前端承担，负责用户交互和推荐结果展示。
- 服务层由 Spring Boot 后端承担，负责业务编排、接口管理、数据库访问和外部服务调用。
- 算法层由 Python FastAPI 服务承担，负责推荐计算。
- 数据层由 MySQL 承担，负责用户、电影、评分和推荐结果存储。

### 1.2.4 智能解释

项目尝试将推荐算法与大语言模型能力结合。推荐算法负责计算推荐结果，大语言模型负责对推荐结果进行语义增强。系统通过后端封装 LLM 调用逻辑，将用户信息、电影信息和预测得分组织为 Prompt，并生成自然语言推荐理由。

## 1.3 研究方法

本项目采用“理论分析、实验验证与工程实现相结合”的研究方法。

首先，通过调研推荐系统基本原理和 MovieLens 数据集结构，明确协同过滤、矩阵分解等算法在电影推荐场景中的适用性。

其次，通过工程实现验证推荐流程。系统将推荐算法封装为独立 Python 服务，由 Spring Boot 后端通过 HTTP 接口调用，使算法实现与业务服务解耦。

最后，通过软件工程方法组织开发过程。团队参考 Scrum 敏捷开发思想，按阶段推进选题、需求分析、架构设计、核心功能实现、系统联调和文档整理。

## 1.4 技术路线

项目技术路线如下：

```text
MovieLens 数据 / 测试数据
        ↓
MySQL 数据库存储
        ↓
Python 推荐算法服务
        ↓
Spring Boot 后端服务
        ↓
Vue 前端展示
        ↓
LLM 推荐理由解释
```

主要技术选型包括：

| 模块 | 技术 |
|---|---|
| 前端应用 | Vue.js / Vite |
| 后端服务 | Spring Boot / Maven |
| 推荐算法服务 | Python / FastAPI / pandas / numpy / scikit-learn |
| 数据库 | MySQL |
| 接口方式 | RESTful API / HTTP JSON |
| 智能解释 | OpenAI-compatible LLM API |
| 协作管理 | Git / GitHub |
| 部署与联调 | PowerShell 脚本 / 宝塔部署脚本 |

## 1.5 可行性分析

### 1.5.1 数据可行性

MovieLens 是推荐系统研究中常用的公开数据集，包含用户评分记录和电影元数据，数据结构规范，适合用于课程项目中的推荐算法建模与实验验证。

### 1.5.2 技术可行性

协同过滤、矩阵分解等推荐算法原理成熟，Python 生态中 pandas、numpy、scikit-learn 等工具能够支持数据处理和算法实现。Spring Boot 能够快速构建稳定的后端服务，Vue.js 能够支持前端页面展示，MySQL 能够满足项目数据持久化需求。

### 1.5.3 工程可行性

团队成员分工覆盖架构设计、后端与算法实现、前端与测试等关键方向。项目采用模块化设计，各模块之间通过标准接口交互，有利于并行开发和后续维护。

## 1.6 项目创新点

本项目的主要创新点包括：

1. 推荐结果可解释。系统不仅返回推荐列表和预测得分，还保存自然语言推荐理由。
2. 推荐计算与语义解释解耦。Python 算法服务负责数值计算，LLM 只作为语义增强层。
3. 架构模块化。系统将前端、后端、算法服务和数据库分层设计，降低模块耦合。
4. 数据链路完整。通过 `recommendations.reason` 字段打通“算法结果 → 语义解释 → 数据库存储 → 前端展示”的链路。
