# 第三阶段全员个人工作总结

## 1. 阶段定位

第三阶段是项目完善与交付准备期，主要目标是围绕用户体验、接口稳定性、测试覆盖、文档完整性和演示可用性进行收口。本阶段重点不是新增大量无关功能，而是让已有功能更稳定、更清楚、更适合课程验收展示。

## 2. 鲍明颉个人工作总结

鲍明颉在第三阶段继续承担项目主导、架构整合、联调验证和文档收口职责，重点推动系统从 MVP 走向可展示版本。

主要工作包括：

1. 重新梳理前端使用流程，将“用户入口”放在页面优先位置，使注册、登录、推荐和评分形成自然操作顺序。
2. 完成 LLM 问答前端入口，使用户能够基于当前推荐结果询问推荐理由或观影建议。
3. 打磨电影搜索与浏览体验，加入分页显示、每页数量选择、上一页/下一页切换和结果总数展示。
4. 修正前端中文展示问题，统一页面文案，使页面更适合课程演示和答辩说明。
5. 增强电影浏览和评分流程，避免用户必须手动输入电影 ID，提高小白用户的可操作性。
6. 补充后端电影分页接口，为前端分页浏览提供稳定的数据结构。
7. 整理第二阶段和第三阶段全员工作总结，突出系统架构、数据库接入、后端链路、前端联调和文档统筹过程。
8. 继续维护 README、接口说明和架构说明，保证其他同学或验收环境能够理解项目如何运行。
9. 对项目剩余风险进行梳理，包括用户安全仍为 MVP、LLM 密钥配置、数据库初始化边界和部署环境迁移问题。

阶段贡献概括：鲍明颉在第三阶段主要完成了前端体验完善、LLM 问答联调、分页浏览设计、后端接口补充、文档总结和交付风险梳理，是项目可展示版本形成的主要推动者。

## 3. 杨舸个人工作总结

杨舸在第三阶段主要围绕服务端稳定性和部署联调继续提供支持。

主要工作包括：

1. 配合确认算法服务和后端服务在本地启动脚本下能够稳定运行。
2. 协助检查推荐接口、评分接口和 LLM 降级接口的返回结构，确保前端能够正常解析。
3. 参与部署脚本和本地测试脚本的维护，保证课程展示时可以快速启动服务。
4. 配合排查后端接口调用问题，包括端口、依赖、数据库连接和服务健康检查。
5. 对算法服务与后端服务的边界保持说明，确保算法层负责计算，后端负责业务封装和解释生成。

阶段贡献概括：杨舸主要保障算法服务、后端服务和部署脚本的稳定性，使项目在完善前端体验时仍能保持服务端链路可用。

## 4. 伍锡飞个人工作总结

伍锡飞在第三阶段主要围绕前端体验、测试反馈和演示流程参与工作。

主要工作包括：

1. 参与前端页面体验检查，关注登录入口、推荐列表、电影浏览和评分提交是否易用。
2. 反馈电影搜索结果数量少、没有页码、评分时不知道电影 ID 等问题，推动前端交互继续完善。
3. 参与手工测试流程复核，确认注册登录、推荐刷新、电影搜索、评分提交和 LLM 问答能够串联展示。
4. 协助准备演示关注点，包括先登录、再看推荐、搜索电影、提交评分、刷新推荐和进行智能问答。
5. 参与整理测试问题清单，为最终验收前的问题收口提供依据。

阶段贡献概括：伍锡飞主要参与用户体验反馈、前端测试复核和演示流程整理，帮助项目从“能运行”进一步走向“能讲清楚、能展示”。

## 5. 第三阶段团队协作总结

第三阶段团队围绕可展示版本进行完善：

- 鲍明颉主导体验打磨、前端联调、接口补充和文档收口。
- 杨舸保障算法服务、后端服务和部署脚本稳定运行。
- 伍锡飞参与前端体验反馈、手工测试和演示流程整理。

本阶段完成后，项目已经具备注册登录、电影浏览、评分提交、推荐刷新和 LLM 问答的完整演示链路。
