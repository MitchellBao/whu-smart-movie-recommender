# 第一阶段全员个人工作总结

## 1. 阶段定位

第一阶段是项目奠基期，主要目标是完成选题确认、团队组建、需求边界梳理、技术路线确定、数据库初步设计、系统架构设计和接口边界确认。本文件只记录阶段个人工作总结，不并入《选题及团队组建报告》《需求规格说明书》或《系统设计说明书》正文。

## 2. 鲍明颉个人工作总结

鲍明颉在第一阶段主要承担组长、Scrum Master、产品负责人和系统架构负责人的职责，是本阶段项目方向、系统边界和工程规范的主要统筹者。

主要工作包括：

1. 完成课程项目小组创建，明确三人团队协作方式、成员职责和阶段推进节奏。
2. 主导项目选题，确定项目主题为“面向大规模用户行为数据的智能电影推荐系统设计与实现”。
3. 依据 Scrum 敏捷开发思想，建立“架构统筹 + 算法后端实现 + 前端测试联调”的协作结构。
4. 主导系统总体架构设计，划分应用层、服务层、算法层和数据层，明确各层职责。
5. 明确推荐算法与大语言模型的职责边界，将 LLM 定位为语义增强与解释生成模块，而不是推荐排序核心。
6. 设计“算法结果 -> LLM 解释 -> 推荐理由持久化 -> 前端展示”的核心数据链路。
7. 主导数据库关键字段设计，尤其在 `recommendations` 表中预留 `reason` 字段，用于保存推荐解释文本。
8. 设计前后端 RESTful API 规范，以及 Java 后端与 Python 算法服务之间的 HTTP JSON 调用方式。
9. 统筹完成第一阶段文档框架，包括选题、需求边界、系统架构、接口草案和阶段计划。

阶段贡献概括：鲍明颉主要完成了项目方向确定、架构方案设计、模块职责划分、接口规范制定、数据库关键设计和文档统筹工作，为后续工程实现提供了统一蓝图。

## 3. 杨舸个人工作总结

杨舸在第一阶段主要承担算法服务、后端服务和部署联调方向的准备工作，重点推进服务端 MVP 的可运行验证。

主要工作包括：

1. 参与推荐算法服务技术路线讨论，确认 Python 算法服务与 Java 后端分离的实现方式。
2. 搭建 `algorithm-service` 算法服务骨架，采用 FastAPI 提供推荐计算接口和健康检查接口。
3. 完成基于样例评分数据的协同过滤与 SVD 推荐流程，用于验证算法层与服务层之间的数据交互。
4. 搭建 Spring Boot 后端 MVP，覆盖推荐、评分、LLM 查询等基础接口。
5. 完成后端分层结构实现，包括 controller、service、repository、entity 和 dto 等模块。
6. 编写本地启动、停止和测试脚本，使后端服务和算法服务具备一键联调基础。

阶段贡献概括：杨舸主要完成了算法服务、后端服务和部署脚本的初步工程实现，使第一阶段架构设计具备可运行验证基础。

## 4. 伍锡飞个人工作总结

伍锡飞在第一阶段主要承担前端与测试方向的准备工作，重点围绕前端工程入口、页面展示方向和接口联调需求进行梳理。

主要工作包括：

1. 参与前端技术路线讨论，确认采用 Vue + Vite 作为前端工程基础。
2. 参与推荐结果展示、评分入口、推荐理由展示等前端功能需求讨论。
3. 配合梳理前端需要展示的字段，包括 `movieId`、`title`、`genres`、`score` 和 `reason`。
4. 参与接口联调关注点整理，明确前端通过后端统一 API 获取推荐数据，不直接调用算法服务。
5. 参与测试关注点整理，包括推荐接口返回结构、评分提交结果、LLM 问答返回结果和服务健康状态。

阶段贡献概括：伍锡飞主要完成了前端方向准备、展示需求梳理和测试关注点确认，为后续前端页面实现和联调提供基础。

## 5. 第一阶段团队协作总结

第一阶段团队已经完成项目主题确定、成员分工、架构方向、工程骨架和基础联调链路。三名成员的工作形成互补关系：

- 鲍明颉负责项目统筹、系统架构、需求边界、接口规范和文档组织。
- 杨舸负责算法服务、后端服务和部署联调脚本的初步实现。
- 伍锡飞负责前端工程准备、展示需求讨论和测试关注点梳理。

本阶段为项目建立了清晰的组织结构和技术基础。
