# 第一阶段个人工作总结

## 1. 阶段定位

第一阶段为项目奠基期，主要目标是完成选题确认、团队组建、技术路线确定、工程骨架搭建、数据库初版设计和核心接口边界确认。本阶段个人工作总结独立记录团队成员贡献，不并入选题报告、需求规格说明书或系统设计说明书。

## 2. 鲍明颉个人工作总结

鲍明颉在第一阶段主要承担组长、Scrum Master 和系统架构负责人的职责，重点完成项目统筹与架构设计工作。

主要工作包括：

1. 完成课程项目小组创建，明确团队协作方式和成员职责。
2. 依据 Scrum 敏捷开发思想，确立“架构统筹 + 算法后端实现 + 前端测试”的三人协作结构。
3. 主导项目选题方向，确定项目主题为“面向大规模用户行为数据的智能电影推荐系统设计与实现”。
4. 主导系统总体架构设计，明确应用层、服务层、算法层和数据层四层结构。
5. 明确推荐算法与大语言模型之间的职责边界，将 LLM 定位为语义增强层，而不是推荐排序核心。
6. 设计“算法结果 → LLM 解释 → 推荐理由持久化 → 前端展示”的数据链路。
7. 主导数据库关键字段设计，特别是在 `recommendations` 表中设置 `reason` 字段，用于保存推荐解释文本。
8. 确立前后端 RESTful API 规范，以及 Java 后端与 Python 算法服务之间的 HTTP JSON 调用方式。
9. 统筹编写与整理第一阶段项目文档，包括选题、需求边界、系统架构和接口设计等内容。

阶段贡献概括：鲍明颉主要完成了项目方向确定、架构方案设计、模块职责划分、接口规范设计、数据库关键设计和文档统筹工作，为项目后续工程实现提供了统一蓝图。

## 3. 杨舸个人工作总结

杨舸在第一阶段主要承担核心算法、后端服务和部署联调实现职责，重点推进项目服务端 MVP 的可运行验证。

主要工作包括：

1. 参与推荐算法服务技术路线设计，确定 Python 算法服务与 Java 后端分离的实现方式。
2. 搭建 `algorithm-service` 算法服务骨架，采用 FastAPI 提供推荐计算接口和健康检查接口。
3. 实现基于样例评分数据的协同过滤与 SVD 推荐流程，用于验证算法层和后端服务层之间的数据交互。
4. 搭建 Spring Boot 后端 MVP，包括推荐、评分、LLM 查询等基础接口。
5. 完成后端分层结构实现，包括 controller、service、repository、entity 和 dto 等模块。
6. 完成 Java 后端调用 Python 算法服务的 HTTP 接口链路。
7. 参与 MySQL 数据库联调，验证用户、电影、评分和推荐结果等核心表结构的可用性。
8. 编写本地启动、停止和测试脚本，使后端服务和算法服务可以一键联调。
9. 完成本地全栈测试链路，使推荐接口、评分接口和 LLM 降级接口可以通过脚本验证。

阶段贡献概括：杨舸主要完成了算法服务、后端服务和部署联调脚本的工程实现，使第一阶段架构设计具备可运行验证基础。

## 4. 伍锡飞个人工作总结

伍锡飞在第一阶段主要承担前端与测试方向职责，重点围绕前端工程入口、页面展示方向和接口联调需求进行准备。

主要工作包括：

1. 参与项目前端技术路线讨论，确定采用 Vue + Vite 作为前端工程基础。
2. 搭建并保留 `frontend` 前端工程骨架，为页面开发和接口接入提供基础入口。
3. 参与推荐结果展示、评分入口和推荐理由展示等前端功能需求讨论。
4. 参与接口联调需求梳理，明确前端需要通过后端统一 API 获取推荐数据，而不是直接调用算法服务。
5. 参与测试关注点整理，包括推荐接口返回结构、评分提交结果、LLM 问答返回结果和服务健康状态。
6. 配合团队确认前端展示字段，包括 `movieId`、`title`、`genres`、`score` 和 `reason` 等。

阶段贡献概括：伍锡飞主要完成了前端工程准备、展示需求梳理和测试关注点确认，为前端页面实现和后续联调提供基础。

## 5. 第一阶段团队协作总结

第一阶段团队已经完成项目主题确定、成员分工、架构方向、工程骨架和基础联调链路。三名成员的工作形成互补关系：

- 鲍明颉负责项目统筹、系统架构和文档规范。
- 杨舸负责算法服务、后端服务和部署联调实现。
- 伍锡飞负责前端工程准备和测试关注点梳理。

本阶段为项目建立了清晰的组织结构和技术基础。
