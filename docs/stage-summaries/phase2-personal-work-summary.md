# 第二阶段全员个人工作总结

## 1. 阶段定位

第二阶段是项目核心功能实现期，主要目标是让“数据库 -> 算法服务 -> Java 后端 -> 前端页面”的主链路跑通，并完成推荐、评分、用户入口、电影检索和基础问答等功能的联调。本阶段重点从设计文档走向可运行系统。

## 2. 鲍明颉个人工作总结

鲍明颉在第二阶段继续承担组长、系统架构负责人和集成负责人职责，重点推进系统从分散模块走向完整链路。

主要工作包括：

1. 统筹后端链路稳定工作，明确项目优先目标为“不依赖前端也能完整返回推荐结果”。
2. 推动 MovieLens 数据集接入 MySQL，确定本地开发环境中数据导入、数据库初始化和服务读取方式。
3. 调整算法服务数据来源，使推荐算法从 MySQL 读取真实评分数据，而不是停留在临时样例数据。
4. 梳理并验证 Java 后端调用 Python 算法服务的流程，保证推荐接口能够返回电影标题、类型、预测分数和推荐理由。
5. 推动用户注册、登录、当前用户保存和评分提交流程的 MVP 实现，解决用户不知道 `userId`、无法长期保存评分的问题。
6. 推动电影搜索与浏览模块实现，使用户可以通过电影名称或类型查找影片，再选择电影进行评分。
7. 设计并完善前端交互流程，将用户入口、推荐结果、电影浏览、评分提交和 LLM 问答组织到同一个可操作页面中。
8. 编写并执行本地全栈测试流程，覆盖 MySQL、算法服务、后端推荐接口、评分接口、用户接口和电影搜索接口。
9. 持续整理 README、接口草案和架构说明，使项目运行方法、数据导入方式和模块职责更加清晰。

阶段贡献概括：鲍明颉在第二阶段主要完成了系统集成、数据库接入、后端链路稳定、前端联调、测试流程和文档更新，是本阶段完整可运行链路的主要推进者。

## 3. 杨舸个人工作总结

杨舸在第二阶段主要围绕算法服务、后端服务和部署脚本继续提供工程支持。

主要工作包括：

1. 参与推荐算法服务结构调整，使算法服务能够按照后端请求返回统一 JSON 数据。
2. 配合完善 FastAPI 算法服务健康检查和推荐计算接口，便于本地脚本检测服务状态。
3. 参与 Spring Boot 后端推荐、评分和 LLM 降级接口的联调，保证基础接口能够被前端调用。
4. 维护本地启动、停止和测试脚本，使算法服务与后端服务可以通过脚本统一启动。
5. 配合排查环境问题，包括 Python 虚拟环境、端口占用、MySQL 连接和依赖安装等。

阶段贡献概括：杨舸主要保障算法服务、后端接口和部署脚本的可运行性，为完整链路联调提供了基础工程支撑。

## 4. 伍锡飞个人工作总结

伍锡飞在第二阶段主要围绕前端页面和手工测试流程参与工作。

主要工作包括：

1. 参与前端页面功能讨论，确认推荐结果、电影搜索、评分入口和用户状态展示的基本布局。
2. 配合检查前端页面与后端 API 的字段对应关系，确认推荐结果和电影列表能够正常展示。
3. 参与手工测试流程梳理，包括注册、登录、刷新推荐、搜索电影、提交评分和再次查看推荐结果。
4. 反馈前端体验问题，例如电影浏览数量偏少、用户不知道电影 ID、缺少页数提示等。
5. 协助确认页面展示效果，为后续前端打磨提供问题清单。

阶段贡献概括：伍锡飞主要参与前端体验反馈、页面联调检查和手工测试流程梳理，帮助项目从“接口可用”进一步走向“用户可操作”。

## 5. 第二阶段团队协作总结

第二阶段团队完成了系统主链路的初步闭环：

- MySQL 保存 MovieLens 用户、电影和评分数据。
- Python 算法服务读取评分数据并生成推荐结果。
- Java 后端统一封装推荐、评分、用户、电影搜索和 LLM 问答接口。
- Vue 前端提供用户入口、推荐展示、电影浏览和评分提交操作。

本阶段的关键成果是系统已经具备本地完整运行和手工测试能力。
